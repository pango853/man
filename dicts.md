				クラウドネイティブのポイントであるマイクロサービスとは、アプリケーションが「サービス」（もしくは「マイクロサービス」）と呼ばれる独立したモジュールに分割されていて、サービスとサービスはそれぞれ独立したプロセスとしてネットワーク経由でAPIを通じて呼び出し合う疎結合の関係になっています。

				アプリケーション開発のmodernかつスマートなアプローチ。マイクロサービス・アーキテクチャ (MSA) は、DevOps における重要な要素の1つです。
				MSA のアプローチでは、1 つのエンタープライズ・アプリケーションを、互いに連携して機能する複数の小規模なコンポーネントへと分割することで、開発と維持を容易にします。各コンポーネントは別々に開発され、それらのコンポーネントが集まることでアプリケーションを形成します。
				これらのコンポーネントは、特定の仮想マシンや物理マシンに依存しません。複数のサーバーやデータセンターをまたいでデプロイすることも可能です。また、マイクロサービスは並行プロセスから隔離された最小の機能単位であるため、スケーリングの実行もレジリエンシー (回復力) の維持も、それぞれ独立しています。たとえコンポーネントの 1 つで機能低下が発生しても、システム全体がそれに引きずられてしまうことはありません。
				マイクロサービスは比較的最近登場したものですが、その注目度は急上昇しています。1,800 名以上の開発者と IT プロフェッショナルを対象に最近実施された調査では、全世界の民間組織のおよそ 70% がすでにマイクロサービスを使用しているか導入を検討中であり、約 3 分の 1 の組織は本番環境で使用していることがわかりました。4マイクロサービスを使用することでソフトウェア開発チームはビジネス成果に注力できるようになり、機能の開発の高速化や品質の向上が可能になります。
				MSA では、たとえば年中無休で稼働する Web ベースのアプリケーションのように、アプリケーションを一連のサービスとして開発および維持しやすくなります。また、市民向けのモバイルアプリ、Web サービス、ポータルも、より高速、スケーラブル、かつ効果的なものを提供できます。
				Google、Amazon、Netflix、Uber など、テクノロジー分野を牽引する企業が日々数多くの動的サービスをデプロイするのにマイクロサービスを使用しているのも当然のことと言えるでしょう。

- Docker
	その重要性: オープンソースのツールである Docker と Kubernetes も、DevOps の主要なコンポーネントです。
	Docker はコンテナの仕様とエンジンのプロジェクトであり、アプリケーションを確実かつセキュアにパッケージ化するための方法を標準化するテクノロジーです。
	それがなぜ重要なのかというと、コンテナが登場する前は、開発者がアプリケーションを作成するときにはハードウェアプラットフォーム (HP、IBM、Dell など) に合わせるか、仮想環境 (Red Hat、VMware、Citrix、Microsoft など) に合わせるしかありませんでした。
	しかし現在では、アプリケーションを作成して Docker 仕様に従ってパッケージ化すれば、Google、Amazon (AWS)、VMware、パブリッククラウド、ベアメタル (ホスト OS ではなくハードウェアに直接インストールされた仮想マシン) など、幅広い環境で実行できます。
	Kubernetes は、Docker をはじめとする Open Container Initiative (OCI) 互換コンテナを複数のホスト上で管理するための業界標準のプロジェクトです。
	2014 年に Google によって公開され、コンテナの大規模な実行に関する 10 年以上にわたる経験を取り入れてきた Kubernetes (ギリシャ語で「航海長」の意) は、ノードのクラスタをまたいで分散された関連コンポーネントを制御する優れた方法を提供します。
	Kubernetes はアプリケーションを構成するコンテナをグループ化して論理単位とするため、管理や検出が容易になります。また、Kubernetes はノート PC、マルチノードクラスタ、パブリッククラウド、オンプレミス、仮想マシン、ベアメタルと、さまざまな環境で実行できます。
